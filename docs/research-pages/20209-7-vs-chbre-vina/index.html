Fall20-W7 Target Virtual Screening - Chembridge Economy (VINA)

Objective#

This is another run using the same dataset as the GOLD screening on the repacked model of FtFBA in week 6.

Methods#

Preparation#

Same as last week, skipped.

Library docking#

The dock was performed on AutoDock Vina 1.1.2 with MGLTools-1.5.6

Preparing Receptor and Ligands#

The receptor was converted into PDBQTformat with the following script:

# docked_prot.pdb is the repacked model input with substrate in place.
echo "centroid resn FBP; save receptor.pdb, polymer;"|pymol -c -p docked_prot.pdb
$AutoDockTools/Utilities24/prepare_receptor4.py -A checkhydrogens -r receptor.pdb

The ligand in library and control was converted in PDBQT format at run time using prepare_ligand4.py (see Running section for details).

Running#

The dock was done on TACC Stampede2 supercomputer.

Sample job worker script:

#!/bin/bash
[ $(cat job.sdf | grep "\$\$\$\$" | wc -l) -eq $(find output -name "*.pdbqt" | wc -l) ] && echo "skipping $PWD" && exit 0
ulimit -u 16384
mkdir input || true
mkdir output || true
find input -name "*.mol2" >/dev/null || obabel job.sdf -p7.4 -m -Oinput/job.mol2
find input/ -type f -name "*.mol2" | xargs -Iligandfn \
bash -c '[ -f "ligandfn.pdbqt" ] || $ADT_SCRIPT_DIR/prepare_ligand4.py -l ligandfn -o ligandfn.pdbqt'
find input/ -type f -name "*.pdbqt" | xargs -P10 -Iligandfn \
bash -c 'ligName=$(obabel $(echo ligandfn | sed "s/\.pdbqt//") -otxt | sed "s/\s/-/g"); [ -f "output/$ligName.pdbqt" ] || vina --config vina.conf --ligand "ligandfn" --out "output/$ligName.pdbqt" --log "output/$ligName.log";'

Score collecting script (VINA .log file to .csv):

#!/usr/bin/awk -f
BEGIN {
print "filename,mode,affinity,rmst_lb,rmsd_ub"
}
BEGINFILE {
tablebegun = 0
}
/\|/ {
tablebegun = 1;
}
/[0-9]/ {
if(tablebegun) {
gsub("\\s+", ",");
print FILENAME $0;
}
}

Discussion#

It was observed that

(1) In terms of classification ability between positive controls and decoys, VINA in this run does provide a better result compared to GOLD last week.

(2) Between the top 10 candidates there are a couple of molecules that have very similar configurations (such as ZINC68844447 and ZINC91306034 and ZINC218226505), which is an indication that VINA does produce similar results for similar configurations.

(3) While the pose produced by VINA for the top 1 candidate does make sense, but a lack of polar contacts or other significant binding motifs indicates that this is not a very optimal pose in terms of binding mode to the receptor.

In terms of limitations, it should be acknowledged that

(1) This dock does not have a very proper evaluation on result precision, although there are traces that indicate precision that are discussed above.

(2) A better performance based on an ROC curve does not necessarily mean better performance in virtual screening, in which case we mostly only care about positives in top10%. Thus, higher selectivity that only shows up laster would not be very useful in virtual screening applications.

(3) This dock also uses rigid receptor configuration. This can be indeed a problem since the model was theoretical thus there can be a bigger deviation between the model of the receptor compared to the actually natural conformation.

Thus, the results from this dock may have higher selectivity compared to the GOLD dock last week, thus I would agree that the results from VINA to be used if the budget for purchasing more compounds was limiting.

Conclusion#

The results from this dock may be used to move on to actual lab screening. However it may be worth considering to find ways to optimize either the model or docking configuration for potentially better results. (See last-week's GOLD discussions for more info.)
